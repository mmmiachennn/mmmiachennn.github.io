爲什麼選擇用 Hugo

當初選擇使用 Hugo 有幾個原因：

  1. 像是 Medium 這種有自己的格式的 blog，一旦遇到文章搬家是一件非常困難的事情，Hugo 是以 Markdown 的方式撰寫，Markdown 是通用格式，相較之下能稍微無痛搬家。
  2. 可以找喜歡的主題或是可以自己建立主題，創造專屬風格的 blog。

Windows 怎麼建立 Hugo？

建議用 WSL 來安裝 Hugo，我試過用 chocolately、scoop 安裝，在 local 執行的時候會一直出問題。

安裝 Hugo 的方法參考此篇文章 ，所需要的工具為 Git、golang、hugo，按照文章步驟進行就可以做出一個與文章主題一模一樣的 blog 出來。

以下爲 Hugo 常用指令

# 檢查 hugo 版本
hugo version

# 建立一個新的 hugo 專案
hugo new site {folder name}

# 在 local 端執行，隱藏 draft
hugo server

# 在 local 端執行，並顯示 draft
hugo server -D 
hugo server --buildDrafts

# 產生正式發布檔 
hugo

如何把 hugo 部署到 github page？

此篇文章有詳細的步驟教學，請留意 hugo 的 config.toml 已經改爲 hugo.toml 囉~

主要步驟：

  1. 建立 github page repo（optional：再建一個 hugo 的 repo）
  2. hugo theme 可以使用 gitsubmodule 的方式 fork
  3. 修改 hugo.toml 的設定，baseURL、theme…等
  4. 執行 hugo server，建立或更新 public 資料夾（產生靜態網站）
  5. 將 public push 到 github page repo

以下爲 git 常用指令

# folder 初始化 git
git init

# 建立 remote repo 關係
git remote add origin https://github.com/../repo

# 檢查 remote 關係
git remote -v

# 移除 remote repo 關係
git remote remove origin

# 建立巢狀 repo
# 在 themes/stack folder 中，建立巢狀 theme_repo（會記錄使用的版次）
git submodule add https://github.com/.../theme_repo themes/stack

# 同步 themes 的最新修改
git submodule update --remote
