Data Modeling

資料建模是資料庫中的資料呈現，實務上會在開發的過程中應應不同的需求，建立多個表來管理與維護。資料建模一部分是爲了後續的業務邏輯與分析，重新將原始資料整理過，來進行分類與管理。

Start Schema

由一個事實表(fact)和多個維度表(dimensional)組成。在星型結構中，事實表會連接到多個維度表，因此資料模型看起來像星形。

以電商資料爲例，商品銷售資訊，串聯不同的維度表，可以計算不同的指標，比如，當月商品購買人數、當月商品賣出數量、MOM 會員購買金額…等。

商品銷售事實表：

  • 產品 ID、User ID、銷售金額

商品維度表：

  • 商品 ID、商品名稱、商品分類 ID

會員維度表：

  • User ID、國家、城市

Snowflake Schema

爲 Start Schema 的延伸，某些維度表可能會再產生一個或多個延伸維度表。當這些維度表放在一起形狀像雪花。

商品維度表：

  • 商品 ID、商品名稱、商品分類 ID

商品分類維度表：

  • 商品分類 ID
  • 分類名稱
  • 商品標籤

Galaxy Schema

結合 Snowflake Schema 與 Star Schema。多事實表的資料模型，也可以稱為 Galaxy Schema。
